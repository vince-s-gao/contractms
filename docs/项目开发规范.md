# 项目开发规范

## 1. 代码规范

### 1.1 前端代码规范

#### 文件命名
- 组件文件：PascalCase，如 `UserInfo.vue`
- 工具文件：camelCase，如 `dateUtils.ts`
- 配置文件：kebab-case，如 `vite.config.ts`

#### 组件规范
```vue
<template>
  <!-- 模板内容 -->
</template>

<script setup lang="ts">
// 导入语句
import { ref } from 'vue'

// 类型定义
interface User {
  id: number
  name: string
}

// 响应式数据
const user = ref<User>({ id: 1, name: '张三' })

// 方法定义
const handleSubmit = () => {
  // 方法实现
}
</script>

<style scoped lang="scss">
/* 样式定义 */
</style>
```

### 1.2 后端代码规范

#### 包结构规范
```
com.contract
├── controller     # 控制器层
├── service        # 服务层
├── mapper         # 数据访问层
├── entity         # 实体类
├── dto            # 数据传输对象
├── vo             # 视图对象
├── config         # 配置类
├── utils          # 工具类
└── exception      # 异常处理
```

#### 命名规范
- 类名：PascalCase，如 `UserController`
- 方法名：camelCase，如 `getUserById`
- 常量：UPPER_CASE，如 `MAX_SIZE`

## 2. 数据库设计规范

### 2.1 表命名规范
- 表名：小写字母，下划线分隔，如 `user_info`
- 字段名：小写字母，下划线分隔，如 `user_name`

### 2.2 索引规范
- 主键：`id` bigint 自增
- 外键：`关联表_id`
- 时间字段：`created_time`, `updated_time`

## 3. API设计规范

### 3.1 RESTful API设计

#### 请求方法
- GET：获取资源
- POST：创建资源
- PUT：更新资源
- DELETE：删除资源

#### URL规范
```
GET    /api/users          # 获取用户列表
POST   /api/users          # 创建用户
GET    /api/users/{id}     # 获取用户详情
PUT    /api/users/{id}     # 更新用户
DELETE /api/users/{id}     # 删除用户
```

#### 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": "2026-03-04 10:00:00"
}
```

## 4. Git提交规范

### 4.1 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

#### 类型说明
- feat：新功能
- fix：修复bug
- docs：文档更新
- style：代码格式调整
- refactor：代码重构
- test：测试相关
- chore：构建过程或辅助工具变动

### 4.2 分支管理

#### 分支命名
- 主分支：`main`
- 开发分支：`develop`
- 功能分支：`feature/功能名称`
- 修复分支：`hotfix/问题描述`
- 发布分支：`release/版本号`
